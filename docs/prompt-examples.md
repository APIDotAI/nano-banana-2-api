# Prompt examples for Nano Banana 2

These prompts are written for testing the Nano Banana 2 API through APIDot. They are starting points for product teams, not guaranteed outputs. Adjust brand names, subject details, source URLs, and safety rules before production use.

## 1. High-volume product variant

Studio product image of a compact travel camera on a brushed aluminum desk with clear engraved text 'LUMA 42', soft morning light, realistic black glass lens, clean editorial composition.

## 2. Ecommerce background swap

Keep the product shape and label from the source image, replace the background with a bright kitchen counter scene, natural daylight reflections, clean ecommerce hero shot.

## 3. Social creative batch

A playful square ad image for a citrus soda can, bold readable label, bright summer table scene, realistic condensation, high-volume campaign variant style.

## 4. Reference-guided layout

Use the source image as the product reference, preserve the package geometry, and create a premium landing-page hero image with clean space on the left for headline text.

## 5. Readable label test

A close-up product photograph of a matte notebook with readable gold text 'FIELD NOTES 2026', soft shadows, natural paper texture, neutral desk styling.

## 6. Marketplace thumbnail

A crisp marketplace thumbnail for wireless earbuds, white background, realistic shadows, front and three-quarter angle, polished but not over-stylized.

## Production prompt notes

- Store prompt versions together with task_id, model, user/job owner, and request payload.
- Keep source media URLs reachable long enough for APIDot to fetch them.
- Be explicit about composition, typography, material, lighting, output size, and reference-image constraints.
- Do not log prompts or source media URLs if they may contain private customer data.
